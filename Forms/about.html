INTRODUCTION 1.1 PURPOSE The purpose of this document is to build an online system to teach users how to train their pets. 1.2 DISCRIPTION OF FORMS AND FORM FIELDS This system uses four forms login, register, personal information signup. A registered user/student can access the personal information page which it takes information about their pets like names last name (required) first name (required) middle name (optional) this page ONLY takes information for pets and the register ONLY takes information for humans. 1.3 PROJECT SCOPE The purpose of the Pet Online Training (POT) is to help people to train their pets properly if they want the pet for example dog: if they want the dog to be a housedog, guard dog, therapy dog, sled dog, search and rescue dog, hunting and flushing dog, mascots etc. 1.4 FORM DEPENDENCIESHow each form depends on the other. The personal information page depends on the registration form. The personal information cannot be accessed and used if you have not been logged in and registered. 1.5 USER CLASS AND CHARACTERISTICSThe admin is the person who can update/edit/change the forms and the web page. The user/student/participant is a user that can use the forms access the web page and learn from it. 1.6 USER STORIES AND TESTING SCENARIOS • A user should be able to signup, login, register, view Animal pages. • A user should be able to use the forms listed above. • An admin should be able to update the pages/forms/site. • An admin should be able to confirm/approve people coming into the online school. 1.7 FUCTIONAL REQUIREMENTS -Placement of elements on the page. -Size of the elements like buttons, pictures, drop-downs, text fields... -Length of the fields like max length, min length, etc... -Fonts, colours, style
